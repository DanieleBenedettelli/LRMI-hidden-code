# LEGO MINDSTORMS Robot Inventor Model Specific Blocks Map

This repository documents how to locate the Python code generated by the black-box model-specific blocks used by the LEGO MINDSTORMS Robot Inventor App hero models.

The goal is to preserve this information from obsolescence and make it easier for anyone who wants to port these blocks to pure Scratch, Pybricks, or another open runtime.

The file `LMRI.js` is provided in this repository as is. It includes the low-level python implementation of such blocks.
The file `LMRI_beautified.js` is the same code but in a more readable format.

## Context

The LEGO MINDSTORMS Robot Inventor App includes several model specific blocks for the official hero robots.

These blocks appear in projects as opcodes (with prefix `flippermodelspecific_`) such as:

```text
flippermodelspecific_bigBotCalibrate
flippermodelspecific_bigBotRemoteControl
flippermodelspecific_bigBotToggleAction
flippermodelspecific_custom
flippermodelspecific_dogBotMoveForSeconds
flippermodelspecific_dogBotSetLegsToPosition
flippermodelspecific_dogBotSetSpeed
flippermodelspecific_dogBotStartMoving
flippermodelspecific_dogBotStop
flippermodelspecific_dogBotWalkForSeconds
flippermodelspecific_moveBotCalibrate
flippermodelspecific_moveBotRemoteControl
flippermodelspecific_playBotCalibrate
flippermodelspecific_playBotDrumBarsAtTempo
flippermodelspecific_playBotRemoteControl
flippermodelspecific_sportBotRemoteControl
```

Inside the bundled JavaScript code, these opcodes are mapped to minified JavaScript functions. Those functions generate the Python code that is eventually sent to the hub.

This README provides the current opcode to JavaScript function map.

## How to Use This Map

1. Search for the function name in the table above.
2. Search the corresponding JS function name in `LMRI.js` or `LMRI_beautified.js`
3. Inspect the generated micropython code string after each `functionBody` tag.
4. Reimplement the behavior in your target environment.

## Notes

Some blocks generate a direct Python helper function call, such as:

```python
dogBotWalkForSeconds(...)
```

Other blocks generate calls to shared helper functions, such as:

```python
bc("A", "E")
bc("B", "A")
bc("C", "A")
```

These shared helpers still need to be inspected and understood separately before a complete port can be written.

## Scope

This repository does not provide a complete port of the LEGO model specific blocks.

It only provides a map to help locate the relevant code inside `LMRI.js`.

`LMRI.js` is included as is, for preservation and research purposes.
